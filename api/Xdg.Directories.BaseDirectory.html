Remarks


This allows users to have less clutter in their home directories, and allows programmers to not have to hard-code directories.


For all base directories, the directories are resolved as follows:
  1. The respective XDG_* enivronment variable will always be used if specified
  2. Windows Use Known Folders.
    macOS Follow the Apple documentation on application-specific files.
    Linux/FreeBSD Follow the XDG Base Directory specification.

ConfigDirs

A list of directories to look for configuration files.

public static IList<string> ConfigDirs { get; }

Property Value

IList<string>

A set of preference-ordered base directories to which configuration files should be searched.

Remarks

If XDG_CONFIG_DIRS is set, a list of the directories specified.

Windows%ProgramData%, %APPADATA%
macOS~/Library/Preferences, /Library/Application Support, /Library/Preferences
Linux/FreeBSD/etc/xdg
This is typically not used, for most use cases ConfigHome should be used instead.
